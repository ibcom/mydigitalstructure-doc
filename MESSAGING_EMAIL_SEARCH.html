  MESSAGING_EMAIL_SEARCH MESSAGING Endpoint
  /ondemand/messaging/?method=MESSAGING_EMAIL_SEARCH  
  Get a list of email (POP/IMAP) messaging accounts for logged on user.
CURRENTLY ONLY XML RETURN FORMAT SUPPORTED.		  
  PARAMETERS TYPE NOTES
  account Numeric ID of an account
  id Numeric Email server message ID.
  start Numeric Logical index to start at, used for pagination.
  rows Numeric Number of rows to return, used for pagination.
  stopatid Numeric Return rows until you reach this email server message ID ie. this can be the last message ID you have and then ask for all news emails. If using IMAP you should consider using the &search= to achieve this.
  folder Numeric IMAP folder ID.
  includeremoved 1 If using the onDemand header cache - this will inclde the ones that have been marked as removed.
  extractstart Text If the message contains some structure data (eg an XML or JSON payload) you then use this to get it returned as seperate field - for extraction and use.
  extractend Text Works with extractstart to mark where to stop to the extraction.
  extractendlast Text Same as extractend, but looks for last instance, not the first to stop extraction.
  onlyincludeifextract 1 If set to 1 it will only include the message in the search return if something is extracted.
  search 1 IMAP search string.
  RETURN  
  id Numeric
  sourcetype Text. 1
  sourcetypetext Text. EMAIL
  to Text. Who the email was sent to. # & | delimeted list with name and email address.
  cc Text. Who the email was sent cced to. # & | delimeted list with name and email address.
  from Text. Name of the person the email was from.
  fromemail Text. Email address of the person the email was from.
  replytomail Text. Reply to email address of the person the email was from.
  date Date. Date sent.
  subject Text
  message Text
  attachmentcount Numeric
  attachments Text. # & | delimited list with name.
  extract Text
  removed 1/0
  read 1/0
