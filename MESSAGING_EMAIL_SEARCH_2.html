  MESSAGING_EMAIL_SEARCH MESSAGING Endpoint
  /ondemand/messaging/?method=MESSAGING_EMAIL_SEARCH  
  Get a list of email (POP/IMAP) messaging accounts for logged on user. These emails are cached on our server allowing for easy querying / manipulation.  
  PARAMETERS TYPE NOTES
  account Numeric Mandatory. ID of the email account
  folder Numeric IMAP folder ID.
  id Numeric Email server message ID. If this parameter is specified, only this message will be retrieved
  includeremoved y/n If using the onDemand header cache - this will inclde the ones that have been marked as removed.
  orderby Numeric By default, it will return the emails in the same order as the email server. If you pass 'date' it will return based on the sent date
  retrieve Numeric If you pass back the 'retrieve' from a previous call, it will only return new emails.
  retrievetype Numeric If passing 'retrieve', 1=Only return new emails if any (default), 2=Return all emails. For efficiency, it is recommended to leave this as the default
  rows Numeric Number of rows to return. Like start, this may be deprecated in the near future
  search Text IMAP search string.
  start Numeric Logical index to start at. Use if you want to ignore some records. May be deprecated in the near future, as pagination is now supported via the 'more' methods
  stopatid Numeric Return rows until you reach this email server message ID ie. this can be the last message ID you have and then ask for all news emails. If using IMAP you should consider using the &search= to achieve this.
  FIELD RETURN  
  skippedemails Numeric. If ignoring remved messages
  unreademails Numeric.
  retrieve Numeric. id in our temporary cache. Can be fed into further messaging methods, eg MESSAGING_EMAIL_ATTACHMENT_SEARCH
  totalcount Numeric.  Total number of emails in inbox.
     
  DATA RETURN  
  contenttype Text. For IMAP only. If this contains multipart/mixed, then it is likely that the email has attachments
  imapflags Text. For IMAP only. Examples include: "(/Seen)", "(/Seen /Deleted)"
  sourcetype Numeric. 1=Email
  sourcetypetext Text
  to Text. Who the email was sent to. # & | delimeted list with name and email address.
  cc Text. Who the email was cc'ed to. # & | delimeted list with name and email address.
  from Text. Name of the person the email was from.
  fromemail Text. Email address of the person the email was from.
  replytoemail Text. Reply to email address of the person the email was from.
  date Date. Date sent.
  subject Text
  attachmentcount Numeric
  attachments Text. Summary of attachments in 'attachmentName|id|attachmentIndex#....' format, where id is our temporary cached id. For a standard grid listing call MESSAGING_EMAIL_ATTACHMENT_SEARCH. Only retrieved is searching for a single message
  removed Text. Y/N
  read Text. Y/N
  message Text. Body of the email. Only retrieved is searching for a single message
  id Numeric. Our internal cached id. Also known as 'row'. Used in further messaging methods
