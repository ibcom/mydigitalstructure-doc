  MESSAGING_CONVERSATION_MANAGE MESSAGING Endpoint
  /ondemand/messaging/?method=MESSAGING_CONVERSATION_MANAGE  
  Add or remove a conversation eg blogs, wikis etc.  
  PARAMETERS TYPE NOTES
  id Numeric Conversation ID.
  title Text If blank, it wil default to current date, time.
  participantcan Numeric [default: 1] 1=Add Posts & Comments, 2=Add Comments Only, 3=View Only, 4=Add Posts Only
  sharing Numeric [default: 1] 1=Added Participants & Network Groups, 2=Everyone (Public), 3=Internal
  object Numeric See CORE_OBJECT_SEARCH.
  objectcontext Numeric  
  remove 1
  alerturl Text (200) Tags:
[[CONVERSATION_ID]]
[[POST_ID]]
[[COMMENT_ID]]
  alertemailfrom Text (100) If not set, messaging@mydigitalstructure.com is used as default.
  alertemailmessage Text (4000) Post Subject Text||Post Text||Comment Subject Text||Comment Text

If the Comment Text is not set then the Post Text is used.

Default text is used if not set, based on conversation &participantcan option.
